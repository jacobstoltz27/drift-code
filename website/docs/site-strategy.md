# Drift Website — Strategy & Design System Reference

The operating document behind the drift marketing site. Everything here reflects
what is actually built in `website/`; update it when the site changes.

---

## 1. Brand strategy

**What Drift is.** A premium social travel platform. People discover places
through real travelers, save experiences, build itineraries, and inspire each
other's journeys. Inside Drift lives **Peregrine**, the AI travel companion
("Ask Peregrine") that removes planning friction through conversation.

**What Drift is not (hard guardrails).**
- Not a booking platform. Never: book / reserve / reservations / hotels or
  flights as inventory / checkout / deals / discounts.
- AI is not the centerpiece. Exploration is the hero; Peregrine quietly clears
  the runway. The falcon appears, it never dominates.
- One paid tier only: **Drift Plus**. No other tiers exist or may be invented.

**Positioning.** Competitors either sell travel (Expedia, Hopper, Airbnb,
Tripadvisor), organize it drily (Wanderlog, Google Travel), inspire without a
path to action (Instagram, TikTok, Pinterest), or lead with AI as a gimmick
(Layla, Mindtrip). Drift's wedge: the only place where *inspiration is
directly actionable* — a community trip becomes your trip in one tap — and
where suggestions are trustworthy *because nothing is for sale*.

**Personality.** Premium, modern, minimal, inspirational, sophisticated,
curious, authentic, worldly, adventurous, human. Never corporate, gimmicky,
overly playful, or futuristic. Feeling: **"The world is waiting."**

## 3. Customer journey map

1. **Stranger** — lands from social/search → Home hero answers "what is this"
   in 5 seconds; imagery sells the feeling. Goal: scroll.
2. **Curious** — Explore/Features/Peregrine pages deepen the story. Goal:
   believe "this solves my planning mess."
3. **Convinced** — Pricing removes cost anxiety (free forever, one honest paid
   tier); FAQs remove the remaining objections. Goal: /waitlist.
4. **Member** — Waitlist page converts with perks (first in line, founding
   status, shape the product) and zero-risk framing (no card, no spam).
5. **Advocate** — Journal + Community pages give shareable substance; the
   invite-to-unlock Free tier turns members into growth.

Every page ends in a conversion surface (WaitlistCTA or contextual CTA).

## 5. Homepage blueprint (why each section exists)

Hero (identity + primary CTA in 5s) → Manifesto (emotional buy-in) → Stats
(social proof early, before feature claims) → Globe/Destinations (the product
is the world; imagery sells movement) → Features (rational justification) →
Peregrine (differentiation without AI-centering) → Steal demo (the "aha"
mechanic, interactive) → Vibe selector (personalization = self-identification)
→ Testimonials (peer proof) → Pricing (cost objection) → FAQ (last objections)
→ Final CTA (the ask, once, big). Order follows AIDA: feel → understand →
trust → act.

## 6. Design system (as implemented)

**Tokens** (`tailwind.config.ts`): `midnight #0A0D12` (bg), `charcoal #151A24`
(surfaces), `ivory #FEFAEF` (text), `golden #38BDF8` (primary accent, legacy
name), `ocean #AFD2FA`, `forest #24356B` (glows/gradients).

**Type**: Fraunces (`font-display`) for logo/headlines/nav/quotes; Inter
(`font-sans`) for body/forms; Plus Jakarta Sans only inside the app-dashboard
mock. Scale: page H1 `text-5xl→7xl`, section H2 `text-4xl→5xl/6xl`, kicker =
`.label-kicker` (golden, 0.24em tracking, uppercase).

**Core components**: `PageShell` (nav+footer wrapper), `PageHero` (kicker /
serif H1 / sub / optional CTA), `Reveal` (scroll fade-rise, 0.8s,
`[0.2,0.8,0.2,1]`), `.glass` card (blurred charcoal, ivory/10 border,
`rounded-3xl p-8`), image cards (`rounded-[24px]`, gradient scrim, hover
`scale-105` over 700ms), pill buttons (golden bg, midnight text, `.magnetic`
hover scale), pill inputs (charcoal bg, golden focus ring), numbered step
cards, quote blocks (Fraunces italic). Every image `background-image` carries
a forest→charcoal gradient fallback so blocked/slow images never leave holes.

**Motion rules**: one entrance per element, `once: true`; durations 0.3–0.9s;
stagger ≤ 0.1s; motion illustrates (steal button check, vibe background
morph), never decorates. Respect content: no parallax on text.

**Dark theme** is the brand (sunrise-over-midnight); there is no light mode by
design.

## 7. SEO

- `metadataBase` + `%s | Drift` template in `app/layout.tsx`; unique title +
  description per page; OG article images on posts.
- `app/sitemap.ts` (priorities favor /, /waitlist, /features, /peregrine) and
  `app/robots.ts`.
- Content clusters: product (features/peregrine/pricing), inspiration
  (explore + journal), trust (about/community/privacy). Journal targets
  long-tail planning queries ("how to plan a trip", "shoulder season").
- All pages statically generated (First Load JS ≈ 87–147 kB shared) for Core
  Web Vitals.

## 8. Accessibility checklist

- Semantic landmarks (`header/nav/main/section/footer`), one `h1` per page.
- Menu button has `aria-label` + `aria-expanded`; search input labelled.
- Ivory-on-midnight and golden-on-midnight exceed WCAG AA for text sizes used;
  `::selection` styled; focus rings on inputs (`focus:ring-golden/20`).
- All interactive elements are real `<a>`/`<button>`; hover states never the
  only affordance. Body copy ≥ `text-sm`, line-height relaxed.
- Known follow-ups: add `prefers-reduced-motion` handling for framer
  animations; add skip-to-content link.

## 9. Technical architecture

Next.js 14 App Router + Tailwind 3 + TypeScript on Vercel (root = `website/`).
Content lives in typed modules (`lib/content.ts`, `lib/blog.ts`,
`lib/searchIndex.ts`) — a deliberate "CMS-lite": one file to edit, type-checked,
zero runtime cost; graduate to a headless CMS only when non-engineers need to
publish. Waitlist: `POST /api/waitlist` → Supabase (`public.waitlist`, RLS
hardened) with graceful local fallback. Search is a static client-side index —
no backend, instant results.

   queries; creator outreach via /creators.
2. **Next** — public destination pages (SEO surface per city), shareable
   itinerary OG cards ("steal this trip" links), referral loop tied to the
   invite-to-unlock Free tier.

   profiles as landing pages; localized versions of top pages.
